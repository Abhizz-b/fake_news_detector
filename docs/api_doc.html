Overview

The AI Fake News Detector API is a service powered by a Large Language Model (LLM) that analyzes news text, extracts core claims, searches for relevant evidence, and returns a verdict on the truthfulness of the claim. The API is built with the FastAPI framework, supports REST requests, and is suitable for integration into a variety of applications.

Basic Information

  • Base URL: http://localhost:8080
  • API Version: 1.0.0
  • Default LLM: Groq (chat/reasoning)
  • Default Embedding Model: Google Gemini (text-embedding-004)

Authentication

The current version of the API does not require authentication to use. The API is primarily designed for local or internal network deployment.

This API uses an LLM to perform fact-checking, ensuring the accuracy and reliability of information. Through automated analysis and evidence retrieval, it helps users identify fake news.

API Endpoints

POST /check

Description: Analyzes the input news text, extracts the core claim, and evaluates its truthfulness based on evidence gathered from a web search.

Request Body

{
  "text": "News text to be fact-checked",
  "api_base": "https://api.groq.com/openai/v1",
  "model": "llama-3.3-70b-versatile",
  "temperature": 0.0,
  "max_tokens": 1000
}

Parameters

Parameter Type Required Default Description
text string Yes - The news text to be fact-checked
api_base string No "https://api.groq.com/openai/v1" Base URL for the LLM API
model string No "llama-3.3-70b-versatile" The language model to use (served via Groq)
temperature float No 0.0 Model temperature (0.0-1.0); lower is more deterministic, higher is more creative
max_tokens integer No 1000 Maximum number of tokens in the response

Response

{
  "claim": "The extracted core claim",
  "verdict": "TRUE/FALSE/PARTIALLY TRUE/UNVERIFIABLE",
  "reasoning": "Detailed reasoning process",
  "evidence": [
    {
      "text": "Evidence text",
      "source": "Evidence source URL",
      "similarity": 0.95
    },
    {...}
  ],
  "timestamp": "2025-05-18T12:34:56.789"
}

Response Fields

Field Type Description
claim string The core claim extracted from the input text
verdict string The verdict: TRUE, FALSE, PARTIALLY TRUE, or UNVERIFIABLE
reasoning string A detailed explanation of how the verdict was reached
evidence array An array of evidence snippets supporting the verdict
evidence[].text string The evidence text content
evidence[].source string The source URL of the evidence
evidence[].similarity float Relevance score between the evidence and the claim (0-1)
timestamp string Timestamp when the response was generated (ISO format)
GET /health

Description: Checks whether the API service is running normally.

Response

{
  "status": "healthy"
}

Usage Examples

Using curl

curl -X POST "http://localhost:8080/check" \
     -H "Content-Type: application/json" \
     -d '{
           "text": "In 2023, China'"'"'s GDP growth rate reached 15%, a record high.",
           "temperature": 0.0
         }'

Python Example

import requests
import json

url = "http://localhost:8080/check"

payload = {
    "text": "In 2023, China's GDP growth rate reached 15%, a record high.",
    "api_base": "https://api.groq.com/openai/v1",
    "model": "llama-3.3-70b-versatile",
    "temperature": 0.0
}

headers = {
    "Content-Type": "application/json"
}

response = requests.post(url, headers=headers, data=json.dumps(payload))
result = response.json()
print(json.dumps(result, indent=2, ensure_ascii=False))

JavaScript Example

async function checkNews() {
    const url = "http://localhost:8080/check";
    
    const payload = {
        text: "In 2023, China's GDP growth rate reached 15%, a record high.",
        temperature: 0.0
    };
    
    try {
        const response = await fetch(url, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(payload)
        });
        
        const result = await response.json();
        console.log(result);
    } catch (error) {
        console.error("Error:", error);
    }
}

Error Handling

When an error occurs, the API returns a standard HTTP error code along with details:

{
  "detail": "Error during fact-checking: specific error message"
}

Common Error Codes

Status Code Description
400 Invalid request parameters or malformed request
500 Internal server error, e.g. the LLM API is unavailable or an exception occurred during processing

Note: Check whether the LLM service is running normally and confirm that the provided API base URL is correct. If deploying with Docker, make sure network connectivity between containers is working.

Implementation Details

Processing Pipeline

  1. Claim Extraction: Uses Groq's LLM to extract the core claim from the input text.
  2. Evidence Search: Uses the DuckDuckGo search engine to retrieve relevant evidence.
  3. Evidence Ranking: Uses Google Gemini's text-embedding-004 model to rank evidence snippets by relevance.
  4. Claim Evaluation: Evaluates the truthfulness of the claim based on the ranked evidence, with output always returned in English regardless of the input language.

Dependencies

  • LLM engine: Uses Groq's cloud API for claim extraction and evaluation
  • Embedding model: Uses Google Gemini's text-embedding-004 for semantic similarity computation
  • Search engine: Uses the DuckDuckGo API to search for relevant evidence

The system uses Groq's hosted LLM and Google Gemini's embedding model, both on free tiers with no credit card required. A low temperature setting (0.0) makes the model more deterministic, which is better suited to fact-checking tasks.

Deployment

Starting with Gunicorn

gunicorn -c gunicorn.conf.py api:app

Performance Considerations

  • Per-request processing time depends on Groq's response speed and web search efficiency
  • Asynchronous processing is recommended for long-running requests
  • By default, the API starts with 4 Gunicorn worker processes; this can be adjusted based on available hardware resources
  • The Streamlit frontend is deployed separately via Streamlit Community Cloud; API keys (Groq, Gemini) should be stored as secrets, never committed to the repo

Limitations

  • The current version relies on an external search engine to retrieve evidence and may be affected by network restrictions
  • Output (claim, verdict, and reasoning) is always returned in English, regardless of the input language
  • Because the system relies on an LLM, verdicts may be subject to the model's inherent biases or inaccuracies
  • Both Groq and Gemini free tiers have rate limits; heavy traffic may result in throttling

Note: The API is designed for internal network use and is not recommended for direct exposure to the public internet. For public access, add appropriate authentication and rate-limiting mechanisms.
